Uso de Script para ModelSim
Arquivos necessários para esta demonstração:
  1. user_logic.vhd
  2. user_logic_TB.vhd
  3. wave.do
  4. load.do

Este script contém instruções para coverage (cobertura de código)

 Passos para a execução desta demonstração:
  1. Salve os 4 arquivos em um diretório qualquer.
  2.  No console do ModelSim (janela Transcript) digite "pwd" para visualizar o diretório  corrente. Digite o comando "cd" para ir para o diretório de trabalho. Exemplo:

    pwd
    # C:/Modeltech_6.1f/examples
    cd z:/cleo_vsim

 Digitando "dir" visualizamos a lista dos 4 diretórios.
  3. Ao contrário do ActiveHDL, que tem que se criar projeto e tudo o mais, no ModelSim é bem mais simples, basta executar o script fornecido, digitando:

       do load.do

    Para que funcione corretamente, deve-se alterar o script load.do para apontar  corretamente o diretório da biblioteca unisim.  Abrir o load.do e alterar a linha "vmap unisim C:/Modeltech_6.1f/unisim"

    Para quem não tem na instalação do ModelSim a biblioteca unisim pode-se buscá-la na cila ou em \\moraes\public.

    Há também em \\moraes\public (e na cila) a última versão do ModelSim (6.1f).

CODE COVERAGE:
  
     Indica a qualidade do test bench escrito, ou seja, por quais linhas do test bench passamos, se as decisões (ifs, cases, whem... - branchs) são tomados ou não, etc.

     É extremamente útil, pois ao escrevermos test benchs muitas vezes não testamos  todo o nosso circuito.
  • na janela "workspace" clique na aba "sim" - temos a hierarquia do projeto. Selecionar o top (user_logic_tb).
  • agora observem na janela "instance coverage" as coberturas. A ULA está coberta em apenas 84.6% nos statments e 83.3% nos braches.
  • Por quê? Selecione a instância alu e analise a janela "missed coverage". Nas abas statment e branch temos as linhas de código não exercitadas pelo test bench.
  • Na aba "files" do "workspace" abra o arquivo user_logic.vhd. Nas linhas 399-420 temos a ULA e seu problemas. As colunas Hits e BC fornecem as estatísticas de hits e branchs, e os respectivos problemas (mais detalhes na documentação).

EXPLICAÇÃO DO SCRIPT load.do LINHA A LINHA:
if {[file isdirectory work]} { vdel -all -lib work } apaga o diretório work se ele existe
#PARA MODELSIM NO PC
vmap unisim C:/Modeltech_6.1f/unisim		 mapeamento da unisim alterar
#PARA MODELSIM NA SUN
#vmap unisim /soft/mentor/modeltech/unisim   		mapeamento da unisim na SUN
vlib work                                             
vmap work work 		definição da biblioteca work
"conecta" o work do VHDL ao diretório work
vcom -cover bcesx user_logic.vhd  
vcom -cover bcesx user_logic_TB.vhd		 compilação dos códigos VHDL
vsim -coverage -t 10ps  work.user_logic_tb  inicia a simulação, com coverage, passo de 10 ps (necessário ao unisim) e ativa o coverage
do wave.do abre a waveform com os sinais que desejamos visualizar,  obtém uma wave.do gravando os sinais de uma simulação
set StdArithNoWarnings 1 comando mágico - remove os irritantes warnings do StdLogicArith
run 2 ms executa por 2 ms (tempo de simulação)
coverage report -file coverage_rep
coverage save  coverag		 geração de arquivos de relatório
